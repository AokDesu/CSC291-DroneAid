# DroneAid — Flutter app

Mobile app for both end users and admins. Single codebase, role-switched at the auth gate.

## First-time setup (Belle, Day 1)

The `pubspec.yaml` declares every package, but platform-specific dirs (`android/`, `ios/`, `web/`, `windows/`, `macos/`, `linux/`) are generated by Flutter, not hand-authored. After cloning:

```bash
cd app
flutter --version          # require 3.22.x or newer
flutter create .           # regenerates platform dirs, keeps existing lib/, test/, pubspec.yaml
flutter pub get
```

`flutter create .` is idempotent — it never overwrites existing files in `lib/` or `test/`.

Then wire FlutterFire:

```bash
dart pub global activate flutterfire_cli
flutterfire configure --project=csc291-drone-relief
```

That writes `lib/firebase_options.dart`. `main.dart` already calls `Firebase.initializeApp()`.

## Run

```bash
flutter run -d <device>          # real device or emulator
flutter run -d chrome            # web (good for early UI work)
```

## Test

```bash
flutter analyze
flutter test
flutter test integration_test/   # once integration tests land
```

## Project layout

```
app/lib/
├── main.dart                          App entry, Firebase init, MaterialApp
├── core/
│   ├── router.dart                    go_router config (every route from §3 of the page-flow doc)
│   └── theme.dart                     Material 3 theme from seed color
├── data/
│   ├── models/                        Pure-Dart models, no SDK imports
│   └── repositories/                  Firebase SDK touchpoints
├── providers/                         Riverpod providers
├── features/
│   ├── auth/                          P-U-01 login, P-U-02 register
│   ├── user/                          P-U-03..09
│   └── admin/                         P-A-01..07
├── widgets/                           Shared widgets (C-NN from the design doc)
└── utils/                             Pure helpers (Thai ID validator, geo math, time fmt)

app/test/                              Unit + widget tests
app/integration_test/                  End-to-end tests (added later)
```
